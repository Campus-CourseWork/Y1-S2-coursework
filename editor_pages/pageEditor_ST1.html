Editor's Page - Student 1

D M P P Dissanayake

Content Overview

Name and Role: I am Student 1, responsible for developing the Splash Screen, Volunteer Page and a Content Page for our Sustainable Cities and Communities website. My role focuses on creating visually engaging and informative pages that align with UN Sustainable Development Goal 11 [1].

Completed Tasks/Pages:

  • Splash Page: A dynamic and engaging introductory page designed for a smooth loading experience with visual effects.
  • Volunteer Page: A comprehensive page for volunteer programs, featuring filtering options, a review submission form and testimonials.
  • Content Page (Green Spaces and Communities): A detailed educational resource page focusing on specific aspects of urban sustainability, structured with an internal navigation sidebar.

Splash Screen

Technical Description

The Splash Page (splash.html) uses a modern, clean design with a focus on visual engagement and a smooth loading experience. Key HTML elements include a <div> for background circles (bg-elements) that provide a dynamic visual effect, a hero section with a logo and title and a loading-section with a progress bar and pulsating dots. CSS (splash.css) leverages flexbox and grid for layout, custom CSS variables for a consistent color palette (e.g., --primary-light, --green-primary) and subtle shadows (--shadow-md). Animations like fadeIn, scaleIn, float, progressAnimation and dotPulse are used for visual flair during loading. The design is responsive, adapting to different screen sizes using media queries.

Accessibility

Accessibility considerations include semantic HTML tags for better screen reader interpretation. Text content has sufficient contrast against background colors. The loading animation provides a visual cue for progress, although a timeout for redirection is commented out, which would need careful consideration for users who might need more time. Interactive elements like navigation links and cards have addEventListener for user interaction, which should also be keyboard navigable. Images use alt attributes for descriptive text.

Volunteer Page

Technical Description

The Volunteer Page (volunteer.html) features an introduction section, a programs section with filtering options, a review submission form and a testimonials section. The layout is structured using a <main> element with a wrapper class. CSS (volunteer.css) utilizes flexbox for the filter container and grid for programs-container and reviews-container to create responsive layouts. Variables from global.css are extensively used for consistent styling, including --neutral-light for backgrounds, --spacing-2xl for padding, and --radius-lg for rounded corners. Buttons and input fields are styled for hover and focus states using transition for smooth visual feedback. The program cards have a uniform image height and subtle hover effects (transform, box-shadow).

Accessibility

Accessibility features include clear label associations for form inputs (e.g., for attribute on labels linked to id on inputs). Interactive elements like buttons and select dropdowns are styled to indicate focus, enhancing keyboard navigation. Error messages for form validation would ideally be programmatically linked to their respective fields using ARIA attributes for screen reader users.

Content Page

Technical Description

The Content Page (content_ST1.html) is designed with a main content area (main wrapper) that employs a flexbox layout to position an internal navigation sidebar (internal-nav) alongside the primary article content (content-article). The internal navigation is sticky for easy access to different sections. CSS (content_ST1.css) defines the layout and styling, using flex: 0 0 250px for the sidebar's fixed width and flex-grow: 1 for the article to fill remaining space. Responsive adjustments are made using @media queries for various screen sizes, modifying grid-template-columns for image grids and case study containers. Styling uses global.css variables for colors (--primary-green-dark, --neutral-white), spacing (--spacing-lg), border-radius (--radius-lg) and shadows (--shadow-md). Internal navigation links have hover and active states for visual feedback.

Accessibility

The page uses semantic HTML5 elements like <aside>, <article> and <nav> to structure content logically, aiding screen readers. The internal navigation provides clear links to sections within the page, improving usability. Images have descriptive alt attributes. The "Go to Top" button enhances navigation. Font sizes and line heights are defined using CSS variables for readability and responsive design ensures content is accessible on smaller screens.

Challenges and Lessons Learned

During the development of these pages, one of the main challenges was ensuring consistent styling across all pages while maintaining responsiveness. This was addressed by leveraging CSS variables defined in global.css for colors, spacing and typography. Another challenge was implementing interactive elements and animations, such as the loading bar and card hover effects, while keeping performance in mind.

Lessons learned include the importance of a well-defined CSS architecture using variables and thorough testing across different devices and browsers to ensure a consistent user experience. Understanding the interplay between HTML structure and CSS properties was also crucial in achieving the desired visual and functional outcomes.

Compliance

The website aims to comply with modern web standards and best practices. HTML5 semantic elements are used for better structure and accessibility. CSS styling adheres to responsive design principles to ensure compatibility across various devices. Efforts have been made to ensure basic accessibility standards are met, such as providing alt attributes for images and clear focus states for interactive elements. Further improvements could include more extensive ARIA attributes and keyboard navigation testing for full WCAG compliance.
