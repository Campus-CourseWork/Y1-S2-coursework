Content Overview

Name and Role: I am Student 2, responsible for developing the Home Page, Table Page, and Content Page (Urban Planning & Smart Cities) for our Sustainable Cities and Communities website. My role focuses on creating engaging, accessible, and informative pages that align with UN Sustainable Development Goal 11 [1].

Completed Tasks/Pages:

  • Home Page: A comprehensive landing page featuring a hero section, mission statement, and interactive content tiles that introduce visitors to sustainable urban development concepts.
  • Table Page: An interactive data comparison page showcasing sustainability metrics for eight major cities, providing valuable insights into urban sustainability performance.
  • Content Page (Urban Planning & Smart Cities): An educational resource page exploring urban planning principles and smart city technologies, with detailed explanations and visual content.

Technical Description

Home Page

The Home Page implements a modern, responsive design using semantic HTML5 elements for optimal accessibility and SEO. The page structure utilizes <main> as the primary content container, with <section> elements organizing different content areas. The hero section employs <h1> for the main heading, ensuring proper document hierarchy and screen reader compatibility.

The hero image is implemented using <img> with descriptive alt text and responsive styling (max-width: 600px, border-radius: 8px). Interactive content tiles are created using <a> elements with class="content-card", enabling both visual appeal and functional navigation. CSS Flexbox (display: flex, flex-wrap: wrap) creates a responsive grid layout that adapts to different screen sizes.

Hover effects are implemented using transform: scale(1.05) and transition: all 0.3s ease to provide visual feedback and enhance user engagement. The color scheme uses CSS custom properties (variables) for consistency, with green tones reflecting the sustainability theme.

Table Page

The Table Page demonstrates advanced table implementation using semantic HTML structure. The <table> element is enhanced with <thead> and <tbody> for proper document structure and accessibility. The table header uses colspan attributes to merge cells for the main title, creating a clean, professional appearance.

Thumbnail images in the header are implemented with <img> elements featuring title attributes for tooltip functionality and alt text for accessibility. The table employs .alt-row CSS class with background-color: #f0f8f0 for alternating row colors, improving readability and visual hierarchy.

External links to data sources use target="_blank" to open in new tabs and aria-label attributes for screen reader accessibility. The sustainability data presented is based on the Arcadis Sustainable Cities Index [2], providing reliable metrics for urban sustainability comparison. CSS properties including box-shadow, border-collapse: collapse, and border-radius create a modern, polished appearance. Interactive hover effects (tr:hover) enhance user experience by highlighting rows on mouse interaction.

Content Page: Urban Planning & Smart Cities

The Content Page implements a comprehensive information architecture using proper heading hierarchy (<h1> through <h3>) for optimal document structure and accessibility. The internal navigation system is built using nested <ul> elements with anchor links (<a href="#section-id">) for smooth scrolling navigation.

Content images are styled with max-width: 600px for responsiveness and border: 1px solid #ddd for visual definition. The "Go to Top" button is implemented using position: fixed for persistent visibility, with background-color: #4CAF50 and border-radius: 50% for modern styling. JavaScript functionality controls button visibility based on scroll position.

The page uses CSS Grid and Flexbox for responsive layout management, ensuring optimal display across devices. Typography is enhanced with line-height: 1.6 for readability and font-weight variations for visual hierarchy. Color contrast ratios meet WCAG 2.1 AA standards [3] for accessibility compliance.

Challenges and Lessons Learned

Challenges:

  • Ensuring responsiveness across devices, resolved by using Flexbox and Grid layouts.
  • Sourcing accurate 2025 data for the Table Page, addressed by estimating based on Arcadis reports.
  • Balancing visual appeal with accessibility, achieved by adding aria-labels and alt text.

Lessons Learned:

  • Semantic HTML improves accessibility and SEO.
  • Modular CSS ensures design consistency.
  • Cross-browser testing is critical for compatibility.

Accessibility

All pages prioritize accessibility through:

  • Semantic HTML: Using <main>, <section>, and <table> following HTML5 specifications [4].
  • Alt Text: Descriptive alt text for images (e.g., "Smart city skyline with digital overlays").
  • Aria-labels: For links and navigation elements.
  • Keyboard Navigation: Ensuring all interactive elements are accessible.
  • Color Contrast: Meeting WCAG 2.1 AA standards [3].

Compliance

The pages comply with Jisc regulations by:

  • Using valid HTML5 and CSS3, verified with the W3C Validator.
  • Ensuring accessibility per WCAG 2.1 guidelines.
  • Providing clear content aligned with SDG 11.
  • Including footer links to editor and validation pages.
