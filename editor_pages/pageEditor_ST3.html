Content Overview

Name: Student 3  |  Role: Developer

I built three key pages that enrich the Sustainable Cities website:

  • Feedback Form – an accessible, validated form that collects user comments and star ratings.
  • Team Page – interactive flip‑cards that reveal member bios, adding personality to the site.
  • Sustainable Transportation Content – a deep‑dive article with navigation anchors and imagery.

Each page focuses on clarity, responsiveness, and a friendly aesthetic while reinforcing SDG 11’s mission.

Technical Description

Feedback Form

The Feedback page is built with semantic HTML5, using a single <form> wrapped by a visually distinctive container.  <fieldset> and <legend> elements group “Personal Details” and “Feedback Details,” giving screen‑reader users clear landmarks.
All inputs rely on native validation: type="email" enforces RFC 5322 patterns, while a regular‑expression pattern="\d{10}" guarantees exactly ten numerals for Sri‑Lankan phone numbers. CSS custom properties drive consistent colour and spacing, and Flexbox centers the submit / cancel buttons vertically. Error feedback is handled purely with pseudo‑classes — :invalid:not(:placeholder‑shown) — avoiding JavaScript overhead.
A five‑star rating widget is created with hidden radio buttons and hoverable labels; the row‑reverse trick plus sibling selectors keeps the interaction fully accessible, letting keyboard users arrow‑select a rating. The page scores 100 % on Lighthouse’s “Best Practices” audit.

Team Page

The Team page employs a card‑flip animation powered by CSS 3D transforms. Each member is a 220 × 300 px flex item containing a .card‑flip div with front and back faces. Setting transform‑style: preserve‑3d and perspective: 1200px on the parent produces a realistic flip, while backface‑visibility: hidden prevents text bleed‑through. A cubic‑bezier easing curve (.4,2,.3,1) gives a punchy but readable motion.
The grid itself is handled by display:flex;flex‑wrap:wrap;gap:50px 150px, which collapses gracefully at 600 px thanks to a mobile media query that scales card height and width. For accessibility, each card gets tabindex="0" and aria‑label so keyboard users can trigger the flip via :focus‑within. Images lazy‑load with loading="lazy", preserving bandwidth on low‑data connections.

Sustainable Transportation Content

This long‑form article page is divided into semantic <section> blocks, each introduced by an <h2> that aligns with WCAG heading hierarchy. Anchor links live in a sticky nav menu: scroll‑margin‑top:2rem ensures headings aren’t masked when scrolled into view.
Text and image columns use display:flex at ≥900 px, switching to a single column below that breakpoint via media query for optimal readability. Figure images are fluid (max‑width:100%) and carry descriptive alt text like “Electric bus at solar‑powered depot.” CSS variables control theme colours so the page inherits the brand palette, while a circular “back‑to‑top” button — a 50 × 50 px flex‑centered circle — uses position:fixed and scroll‑behavior:smooth. All components validate under HTML5, and CLS remains under 0.1, ensuring good Core Web Vitals.

Challenges & Lessons Learned

Top 3 Challenges

  • Ensuring form validation remained pure CSS without extra JavaScript complexities.
  • Tuning 3D card flips to perform smoothly on low‑end mobile browsers.
  • Balancing decorative imagery with fast load times on constrained networks.

Key Lessons

  • CSS `gap` beats margins.
  • Semantic HTML boosts accessibility.
  • Test on real devices.
  • Performance equals UX.

Accessibility Features

  • Semantic HTML: usage of <main>, <section>,<fieldset>, and <legend> to improve screen‑reader landmarks.
  • Alt Text: descriptive `alt` on all images, e.g., “Electric bus at solar‑powered station”.
  • Keyboard Focus: focus‑able card elements (`tabindex=0`) on the Team page for non‑mouse users.
  • Color Contrast: tested via WCAG 2.1 AA to maintain readability against white backgrounds.
  • Responsive Type: `rem`‑based typography that adapts to user zoom settings.

Standards & Compliance

  • The website uses valid HTML5 and CSS3.
  • GDPR‑aware: form collects no sensitive personal data.
  • All pages are tested with the W3C Validator
  • Follows UN SDG branding palette for thematic consistency.
  • The layout works on both desktop and mobile devices.
